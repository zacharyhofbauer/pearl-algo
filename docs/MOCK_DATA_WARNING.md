# Mock Data Testing - Important Notes

## ⚠️ These Tests Use SYNTHETIC Data

All test scripts (`test_signal_generation.py`, `test_nq_agent_with_mock.py`, `validate_strategy.py`) use **mock/synthetic data** generated by `MockDataProvider`.

### What These Tests Validate

✅ **Strategy Logic** - Signal generation algorithms work correctly  
✅ **Data Processing** - Data fetching and processing pipelines function  
✅ **Service Components** - All service components initialize and run  
✅ **Error Handling** - Error handling and recovery mechanisms work  
✅ **Telegram Integration** - Notifications are sent correctly  

### What These Tests Do NOT Validate

❌ **Real Market Performance** - Prices are synthetic, not real market data  
❌ **Actual Trading Signals** - Signals are based on fake data patterns  
❌ **Live Market Conditions** - No real market regime detection  
❌ **Real Risk Metrics** - Risk calculations use fake prices  

## Mock Data Details

### Default Parameters

- **Base Price:** ~17,500 (realistic NQ futures typical range: ~17,000-20,000)
- **Volatility:** 25-50 points per bar (synthetic movement)
- **Trend:** Configurable (for testing different scenarios)

### Price Generation

The mock data provider generates:
- Random price movements based on Gaussian distribution
- Synthetic OHLC bars
- Fake volume data
- No correlation to real market conditions

## Testing Real Market Performance

To test with **real market data**, you need to:

1. **Use IB Gateway with Live Data**
   ```bash
   # Start IB Gateway
   ./scripts/start_ibgateway_ibc.sh
   
   # Start agent with real data provider
   ./scripts/start_nq_agent_service.sh
   ```

2. **Monitor Real Signals**
   - Watch Telegram for actual signals
   - Check logs: `tail -f logs/nq_agent.log`
   - Review performance metrics

3. **Paper Trading**
   - Test with real data but no actual orders
   - Validate signal quality over time
   - Track performance metrics

## Why Use Mock Data?

Mock data testing is valuable because:

1. **Fast Testing** - No need to wait for market hours
2. **Deterministic** - Same inputs produce same outputs
3. **Controlled Scenarios** - Test specific market conditions
4. **No External Dependencies** - Works without IB Gateway
5. **Safe Testing** - No risk of accidental trades

## Summary

- ✅ **Mock data tests** = Validate code logic and components work
- ✅ **Live data tests** = Validate strategy performance with real market
- ⚠️ **Never trade** based on mock data signals

Always validate strategy performance with real market data before using in production!
